Text Mining in Biomedicine

txtfnnl

My current main project, a modular text mining pipeline for biomedicine using the UIMA framework. It provides a set of Annotation Engine (AE) modules that can be combined to run more complex text mining tasks such as Gene Normalization (assigning DB IDs to gene symbols) or mining for biological events (e.g., transcription regulation interactions). My framework adds a special builder pattern implementation to make UIMA/uimaFIT AEs and Resources less of a chore to develop, maintain, and configure. In addition, my framework provides a "type-free" "one-model-fits-all" type system, that is a single text annotation type with an annotator@namespace:identifier schema to make any kind of annotations on a UIMA SOFA. Last it has special wrappers for the Apache Tika project (a libary to extract text content from many different file-types) that add functionality beyond the raw extraction alone.

medic

A command-line tool to manage an up-to-date PubMed/MEDLINE DB mirror. This is a Python3 program that allows you to bootstrap a MEDLINE Citation (PubMed Abstracts) repository in either SQLite or PostgreSQL (Technically, it uses SQL Alchemy, so most datastores should be OK.) You can import data directly from NCBI eUtils or by parsing MEDLINE XML files. Content can be extracted in plain-text (full, TIAB only, or as TSV) or HTML formats. This is an evolution of the libfnl medline tool. The easiest way to install it is from PyPi, using "pip3 install medic". To use it, check out "medic --help" and "man medic".

gnamed

A command-line tool to maintain an up-to-date database of gene and protein names together with their most important keywords (kDa, length, chromosome position, etc.) and their references to the NCBI taxonomy and to PubMed. This is very useful to build gene normalization systems. Beyond just duming the data, this tool groups unique genes and proteins from different databases and links proteins and genes (as a n to m relation). To use it, clone it from GitHub ("git clone git@github.com:fnl/gnamed.git") and install it locally ("virtualenv-3.3 gnamed; cd gnamed; . bin/activate; python3 setup.py install"). Then, you can learn about it by using man gnamed (or just visit the link...).

libfnl

My continuous Python library for supporting all types of BioNLP tasks I am interested in: gnamed is a library to import gene and protein symbols, names, keywords, and references into a unified PostgreSQL repository. medline is a tool to maintain a on-site instance of the NLM MEDLINE database. Several other minor tools and libraries are available, such as annotator evaluation or interfacing to CouchDB with my own Python 3k client (written a few years ago when there was no other Py3k CouchDB client around). Essentially it is the "source" of the two projects just described above.

BioCreative

An international project organizing community challenges in text mining for molecular biology. I have in particular been the main organizer of one of these challenges, BioCreative II.5, and have developed a Django-based website to manage these events and their participants. Also part of this project is my bceval evaluation script to evaluate the perfomance of text mining systems.
