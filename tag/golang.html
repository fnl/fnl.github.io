Getting started with a "virtual" Go environment (on POSIX)

Given how easy it is to write highly concurrent code in Go (aka "golang"), it is probably worth learning this language. Personally, I believe Go is not yet mature enough for "production" projects other than servers maybe (and I am sure there are people who will not agree with my belief...). But for doing science, juggling huge ammounts of data (no, I will not say the "B"-word...), and your typical scripting pipelines, Go is really great. On top of that, Go is really easy to learn because it has a simple syntax and has a very "bare bones" approach in so many aspects. So if you feel like giving it a try, here is a quick cookbook recipe to bootstrap a Go development environment within moments:

First, install Go itself using your package manager; For example, on OSX, you might use Homebrew, and on Ubuntu or Debian you'd probably use apt:

brew install go --cross-compile-common
apt-get install golang
...

Notice that if you are on a LTS version of Ubuntu (Precise/12.04 right now) or Debian, you might actually want to install a more up-to-date binary. Next, you need to set the global environment variable GOROOT. It has to point to the directory you installed Go into; When using Homebrew on OSX, this can be tricky, so here is a snippet that will create the correct path for you:

export GOROOT=$(brew --prefix)/Cellar/go/$(go version | cut -f3 -d' ' | sed 's/go//')/libexec

Protip: If in the next steps you see a lot of errors of the form:

imports PKGNAME: unrecognized import path "PATH"

They occur when running go get PKG, and it most likely just means that your GOROOT is wrong or not set. Finally, you want to quickly bootstrap a "virtual" development environment for Go. For this, I use a simple shell script (that I call goinit) to set up the directory structure and an "activate-able" environment:

#!/bin/sh

# setup a directory structure for programming in go

mkdir -p "$1"
cd "$1"
mkdir -p src/github.com/fnl
mkdir bin
mkdir pkg
# exmple if you want default packages pre-installed in your environments:
GOPATH=`pwd` go get github.com/nsf/gocode
echo export GOPATH=`pwd` > bin/activate
echo export PATH=\"\$GOPATH/bin:\$PATH\" >> bin/activate
echo export PS1=\"\(`basename "$1"`\)\$PS1\" >> bin/activate

Put this script somewhere on you PATH and make it executable (chmod 755 goinit). Finally, you will want to start the Go Tour to learn about the language itself (highly recommendable!). Ensure you have the GOROOT set in your environment and goinit on your path; Bootstrapping your next Go project (here, the "gotour" as an example) now is as simple as:

goinit path/to/my/work/directory/gotour
cd path/to/my/work/directory/gotour
source bin/activate
go get code.google.com/p/go-tour/gotour
gotour

The "gotour" should open in your browser. Happy Go coding!
